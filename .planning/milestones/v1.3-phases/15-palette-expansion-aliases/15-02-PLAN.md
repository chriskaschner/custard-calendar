<objective>
Create FLAVOR_ALIASES dict with ~20 alias mappings, wire alias resolution into getFlavorProfile(), and add CI validation tests so duplicate/variant flavor names resolve to canonical profiles.

Purpose: Phase 16 will create FLAVOR_PROFILES for ~107 unprofiled flavors. Many Culver's flavors have variant names (historical renames, brand-specific names). Without alias resolution, either (a) duplicate profile entries are needed (maintenance burden) or (b) the variant name falls through to keyword fallback (wrong profile). FLAVOR_ALIASES eliminates both problems.
Output: FLAVOR_ALIASES in flavor-colors.js + cone-renderer.js, modified getFlavorProfile with alias step, CI test, alias visibility in flavor-audit.html.

function normalizeFlavorKey(flavorName) {

    .trim();

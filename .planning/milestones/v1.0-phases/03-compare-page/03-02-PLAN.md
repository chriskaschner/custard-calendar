<task type="auto">
  <name>Task 1: Add accordion expand to store rows</name>
  <files>custard-calendar/docs/compare-page.js</files>
  <action>
    Add accordion expand logic to compare-page.js inside the IIFE. The detail panel HTML was stubbed in Plan 01 (hidden div inside each .compare-store-row). This task wires the click handler and populates content.

    1. **toggleExpand(rowEl):** Click handler for .compare-store-row.
       - If rowEl === _expandedRow, collapse it (set detail hidden=true, clear _expandedRow)
       - If _expandedRow exists, collapse the old one first (synchronous, set hidden=true)
       - Expand rowEl: populate .compare-store-detail content, set hidden=false, set _expandedRow=rowEl
       - Use hidden attribute (boolean, instant) NOT CSS transitions -- prevents rapid-click race condition per research pitfall #3

    2. **populateDetail(detailEl, slug, dateStr):**
       - Get flavor data: _storeData[slug].flavors schedule for dateStr -> {title, description}
       - Get rarity data: only for today's date from _storeData[slug].today -> rarity object
       - Build detail HTML:
         - Flavor description paragraph
         - Rarity detail (if today and rarity exists): "Shows up roughly every N days" or "Ultra Rare -- only every N days!"
         - Google Maps directions link: https://google.com/maps/dir/?api=1&destination=ENCODED_ADDRESS
         - Address from _storeManifest[slug] (address + city + state)
       - Use escapeHtml() for all user-facing text

    3. **Wire click handler:** In renderGrid(), add click event listener to each .compare-store-row that calls toggleExpand. Store the date string on each row with a data-date attribute so populateDetail knows which day's data to show.

    4. **Prevent expand on excluded rows:** If the row has .compare-excluded class, the click handler should return early (CSS pointer-events:none handles most cases, but belt-and-suspenders).

    Important: Use var (not const/let), function expressions, no arrow functions per project convention.
  </action>

  <action>
    Add exclusion filter chip rendering and dimming logic to compare-page.js, and refine filter CSS in style.css.

    1. **EXCLUSION_CHIPS constant** (inside IIFE):
       ```
       var EXCLUSION_CHIPS = [
         { key: 'mint', label: 'No Mint' },
         { key: 'chocolate', label: 'No Chocolate' },
         { key: 'caramel', label: 'No Caramel' },
         { key: 'cheesecake', label: 'No Cheesecake' },
         { key: 'peanutButter', label: 'No Peanut Butter' },
         { key: 'pecan', label: 'No Nuts' },
       ];
       ```
       These 6 chips cover dietary-relevant and allergy-relevant families. Skip turtle (overlaps chocolate+caramel), cookie (too broad), berry (rarely excluded).

    2. **renderFilterChips():** Create a .compare-filter-bar div with one .compare-filter-chip button per EXCLUSION_CHIPS entry. Insert above #compare-grid in the DOM. Each chip has data-family attribute matching the key. Restore active state from localStorage 'custard-exclusions' on render.

    3. **_exclusions** (private Set): Tracks active exclusion family keys. Initialized from localStorage 'custard-exclusions' (JSON array) on init.

    4. **toggleExclusion(familyKey):** Add/remove from _exclusions Set. Toggle .active class on the chip button. Save _exclusions to localStorage as JSON array. Call applyExclusions().

    5. **applyExclusions():** Query all .compare-store-row elements. For each, read data-flavor attribute, call CustardPlanner.getFamilyForFlavor(flavor). If the family is in _exclusions, add .compare-excluded class. Otherwise remove it. If an excluded row is currently expanded (_expandedRow), collapse it.

    6. **Wire chip click handlers:** In renderFilterChips(), add click handler to each chip that calls toggleExclusion(chip.dataset.family).

    7. **Call renderFilterChips() from renderGrid()** -- chips appear above the grid only when grid is shown (not in single-store empty state).

       - Add focus-visible style for accessibility on filter chips

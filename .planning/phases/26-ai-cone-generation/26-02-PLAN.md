**1. Update generateImage() function:**
```javascript
async function generateImage(apiKey, prompt, { model = 'gpt-image-1', quality = 'medium', size = '1024x1024' } = {}) {
  const body = {
    model,
    prompt,
    n: 1,
    size,
    quality,                    // 'low', 'medium', or 'high'
    background: 'transparent',  // Native transparent bg
    output_format: 'png',       // Required for transparency with gpt-image-1
  };

  // For dall-e-3 backward compat (remove after full migration)
  if (model === 'dall-e-3') {
    delete body.background;
    delete body.output_format;
    body.response_format = 'b64_json';
  }

  const res = await fetch('https://api.openai.com/v1/images/generations', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${apiKey}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(body),
  });

  if (res.status === 429) {
    const retry = parseInt(res.headers.get('retry-after') || '60', 10);
    throw Object.assign(new Error('rate_limited'), { retryAfter: retry });
  }
  if (!res.ok) {
    const text = await res.text();
    throw new Error(`API ${res.status}: ${text}`);
  }

  const json = await res.json();
  if (json.data[0].b64_json) {
    return Buffer.from(json.data[0].b64_json, 'base64');
  }
  const imgRes = await fetch(json.data[0].url);
  return Buffer.from(await imgRes.arrayBuffer());
}
```

Create `tools/postprocess_ai_cones.mjs` -- a sharp-based pipeline that processes raw AI-generated PNGs from `docs/assets/ai-candidates/` into final optimized output.

**CLI interface:**
```
node tools/postprocess_ai_cones.mjs                          # process all candidates
node tools/postprocess_ai_cones.mjs --flavor vanilla         # process one flavor
node tools/postprocess_ai_cones.mjs --output-dir docs/assets/cones  # final output (after QA)
node tools/postprocess_ai_cones.mjs --width 288 --height 336       # custom dimensions (default)
node tools/postprocess_ai_cones.mjs --width 144 --height 168       # match existing
```

**Default behavior:** Read all PNGs from `docs/assets/ai-candidates/{slug}/*.png`, post-process each, write to `docs/assets/ai-candidates/{slug}/processed/{original-name}.png`.

When `--output-dir` is specified, write processed files there using `{slug}.png` naming (for final deployment after QA selects a winner).

**Sharp pipeline per image:**
```javascript
await sharp(inputPath)
  .trim({ threshold: 5 })           // Conservative trim of transparent edges
  .resize({
    width: 288,                       // 2x of 144 for retina (default)
    height: 336,                      // 2x of 168 for retina (default)
    fit: 'contain',                   // Preserve aspect ratio
    kernel: 'nearest',               // Preserve pixel art crispness
    background: { r: 0, g: 0, b: 0, alpha: 0 },  // Transparent padding
  })
  .png({
    compressionLevel: 9,             // Maximum lossless compression
    palette: false,                   // Keep full RGBA for transparency
  })
  .withMetadata({ density: 300 })    // Match existing DPI convention
  .toFile(outputPath);
```

**Import sharp** using the same fallback pattern as generate-hero-cones.mjs (try direct import, then worker/node_modules createRequire fallback).

**Progress output:** `[N/total] processing {filename} -> {output} (WxH, Xkb)` per file. Summary at end.

**Validation step:** After processing each image, read it back with `sharp(outputPath).metadata()` and verify: width matches target, height matches target, channels === 4 (RGBA), format === 'png'.

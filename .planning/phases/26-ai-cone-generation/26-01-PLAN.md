**Rules for writing premium overrides:**
1. `base`: Always reference the hex color from BASE_COLORS. Add marbling/cream streak details specific to the flavor. For vanilla base, mention "warm ivory-white (#F5DEB3) with subtle cream highlights". For chocolate, "deep cocoa (#6F4E37) with darker swirl undertones". For mint, "cool mint green (#2ECC71) with cream-white marbling".
2. `swirls`: If profile has a ribbon, describe it with the RIBBON_COLORS hex. Add visual detail: "visible through inclusions", "glossy", "thin drizzle pattern". If no ribbon, write "None -- pure {base} with {density description}."
3. `chunks`: Describe each topping individually with its TOPPING_COLORS hex. Give each topping a distinct visual treatment: OREO gets "visible cream filling layers", pecans get "warm toasted ridges", cookie dough gets "rounded dough balls with chocolate chip flecks", heath gets "golden shard-shaped brittle pieces". When multiple toppings, separate with semicolons.
4. `texture`: Tie it to the density value from the profile. Pure = "Smooth, luminous scoop surface". Standard = "Balanced chunk placement with clear depth". Double = "Generous primary inclusion emphasis". Explosion = "Dense layered inclusion field". Overload = "Saturated single-ingredient coverage". Always add a sentence about ingredient readability.

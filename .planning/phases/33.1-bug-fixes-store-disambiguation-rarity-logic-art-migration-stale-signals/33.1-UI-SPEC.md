## Typography

| Role | Size Token | Computed | Weight | Line Height | Usage in this phase |
|------|-----------|----------|--------|-------------|---------------------|
| Display | `--text-2xl` | 28px (1.75rem) | 700 | 1.2 | Hero card flavor name (unchanged) |
| Body | `--text-base` | 14px (0.875rem) | 400 | 1.4 | Store chip names, flavor row labels, hero card footer store name |

## Component Inventory

### Modified Components

#### 1. Store Name Display (all surfaces)

**Change:** Replace raw `store.city` with disambiguated name from `getDisplayName(store)` utility.

**Format:** `"{slug-derived street name} - {city}, {state}"` for stores in cities with multiple locations. Single-store cities retain `"{city}"` only.

**Affected elements:**

| Surface | CSS Class | Current display | New display |
|---------|-----------|----------------|-------------|
| Compare store chip | `.compare-store-chip-name` | "Madison" | "Mineral Point Rd - Madison" |
| Compare flavor row | Store label in grid | "Madison" | "Mineral Point Rd - Madison" |
| Hero card footer | `.today-store` | "Madison" | "Mineral Point Rd - Madison" |
| Near-me card | `.near-me-card-store` | "Madison" | "Mineral Point Rd - Madison" |
| Rarity banner text | Removed (see item 3) | n/a | n/a |

**CSS addition for chip truncation:**
```css
.compare-store-chip-name {
  max-width: 180px;
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

#### 2. Rarity Badges (Today + Compare)

**Change:** Tighter thresholds, data quality gate, network-wide gate.

**Visual rules (no CSS changes, logic-only):**
- Ultra Rare: avg gap >150 days, min 10 appearances over 90+ days, <100 stores in last 30 days -- badge uses `.rarity-badge-ultra-rare`
- Rare: avg gap 90-150 days, same quality + network gates -- badge uses `.rarity-badge-rare`
- All other flavors: no badge displayed

**CSS addition for uncommon suppression:**
```css
.rarity-badge-uncommon { display: none; }
```

#### 3. Rarity Banner Removal (Compare page)

**Change:** Remove the `buildRarityNudge()` top-of-page banner entirely.

**Removed element:** The full-width rarity nudge banner that duplicated row-level rarity badges.

**Visual impact:** Compare page loads directly into the store selection UI without an interstitial rarity callout. Row-level `.rarity-badge` inline badges remain the single rarity signal per flavor.

#### 4. Week-Ahead Cone Art Migration

**Change:** Replace `renderMiniConeSVG()` with `renderHeroCone()` in week-ahead cards.

**Before:** L0 SVG inline cones, constrained to 27x30px by `.cone-sm svg` (line 1755).

**After:** L5 PNG loaded by `renderHeroCone()` with automatic L0 SVG fallback on error. CSS downscales the PNG to fit the week-ahead card.

**CSS addition for week-ahead PNG sizing:**
```css
.week-day-cone img {
  width: 40px;
  height: auto;
  display: block;
  margin: 0 auto;
}
```

The `img` selector targets the `<img>` element created by `renderHeroCone()` for PNG loading. The SVG fallback continues to be constrained by `.cone-sm svg`. Width of 40px provides a visible upgrade from the 27px SVG while fitting within the 140px-wide `.week-day-card`.

#### 5. Insight Card Filtering (Today page)

**Change:** DOW pattern insight card only renders if the signal's flavor matches today's actual FOTD at the user's store.

**Visual impact:** When today's flavor has no DOW signal, the insight section is empty and hidden. No "peaks on Fridays" card appears for an irrelevant flavor.

**Interaction:** No visual change to the card itself -- same `.signal-card` styling with `.card--accent-sm` border-left. The change is purely a visibility gate.

#### 6. Widget Cone Art Fix (Scriptable)

**Change:** Debug and fix `getConeImage()` PNG loading failure; improve `drawConeIcon()` DrawContext fallback.

**Visual contract for DrawContext fallback improvement:**
- Fallback cone must use a triangular waffle cone shape (not a circle/pin shape)
- Cone body color: use the flavor's base color from `BASE_COLORS` lookup
- Scoop: single circle atop the cone triangle
- Maximum canvas size: 50x50pt (Scriptable DrawContext constraint)
- No text labels in the fallback -- pure shape art

Source: CONTEXT.md widget art fix decision + `widgets/custard-today.js` existing code.

## Interaction Contract

### Store Name Disambiguation

No interaction changes. The disambiguated name is a display-only change. Store selection still operates by slug (internal identifier), not by display name.

### Rarity System

```
[API returns rarity data]
    |
    +-- Gate 1: appearances >= 10 AND span >= 90 days?
    |     NO  --> No rarity label. Badge hidden.
    |     YES --> Continue to Gate 2
    |
    +-- Gate 2: network_store_count > 100 in last 30 days?
    |     YES --> Suppress rarity label. Badge hidden.
    |     NO  --> Continue to Gate 3
    |
    +-- Gate 3: avg_gap_days > 150?
    |     YES --> "Ultra Rare" badge
    |     NO  --> avg_gap_days >= 90?
    |               YES --> "Rare" badge
    |               NO  --> No badge
```

### Compare Page Rarity UX

```
[Compare page loads]
    |
    +-- buildRarityNudge() call REMOVED
    +-- Individual flavor rows render with inline .rarity-badge (if gates pass)
    +-- Accordion detail text retained for curious users
```

### Insight Card Visibility

```
[Signals API returns DOW pattern signal]
    |
    +-- Signal flavor === today's FOTD at user's store?
    |     YES --> Render signal card (existing signalCardHTML)
    |     NO  --> Do not render. Section hidden if no cards remain.
```

### Week-Ahead Art Loading

```
[renderWeekStrip() builds each day card]
    |
    +-- Call renderHeroCone(day.flavor, coneContainer, fallbackScale)
    |     PNG loads? --> Display L5 PNG at 40px width
    |     PNG fails? --> renderMiniConeSVG fallback at 27x30px (existing .cone-sm svg)
```

No new transitions or animations. All changes are data-driven visibility and content changes within existing card patterns.

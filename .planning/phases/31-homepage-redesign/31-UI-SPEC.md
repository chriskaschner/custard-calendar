## Spacing Scale

Declared values use the existing `--space-*` CSS custom property tokens. All values are multiples of 4px.

| Token | CSS Variable | Value | Usage in this phase |
|-------|-------------|-------|---------------------|
| xs | `--space-1` | 4px (0.25rem) | Rarity badge internal padding, icon gaps |
| sm | `--space-2` | 8px (0.5rem) | Gap between flavor name and description, skeleton line margin |
| md | `--space-3` | 12px (0.75rem) | Hero card internal element gaps, week strip card padding |
| lg | `--space-4` | 16px (1rem) | Body padding, hero card side padding, first-visit prompt padding |
| xl | `--space-5` | 24px (1.5rem) | Hero card vertical padding, section margin below fold |
| 2xl | `--space-6` | 32px (2rem) | Header top padding |

Exceptions:
- **12px (`--space-3`):** Grandfathered exception. This token predates the 8-point standard set and is defined in the existing `docs/style.css` `:root` block (line 23). Replacing it with 8px would break existing card layouts site-wide; replacing with 16px would create excessive internal gaps in compact cards. Retained for backward compatibility with the established token system.
- Hero card left accent border: 3px (existing `.today-card` pattern, not a spacing token)
- CTA touch targets: minimum 44px hit area (existing `.btn-text` pattern with `min-height: 44px`)

Source: Pre-populated from existing `docs/style.css` `:root` tokens (lines 21-27).

## Typography

All sizes reference existing CSS custom property tokens. Two weights used: 400 (regular) and 700 (bold). No other weights are permitted in this phase.

| Role | Size Token | Computed | Weight | Line Height | Usage in this phase |
|------|-----------|----------|--------|-------------|---------------------|
| Display | `--text-2xl` | 28px (1.75rem) | 700 | 1.2 | Hero card flavor name |
| Body | `--text-base` | 14px (0.875rem) | 400 | 1.4 | Flavor description, CTA line, first-visit prompt body, button text |
| Label | `--text-sm` | 12px (0.75rem) | 700 | 1.2 | Week strip day names, store name in hero meta, rarity badge text, freshness timestamp, week day dates |

Button text: 14px (`--text-base`) at weight 700 -- distinguished from body copy by weight, not size.

Source: Pre-populated from existing `docs/style.css` `:root` tokens (lines 11-19) and `.today-flavor-name` (line 1903), `.today-flavor-desc` (line 1911).

## Color

The existing color token system defines the full palette. This phase uses the established 60/30/10 split.

| Role | Token | Value | Usage in this phase |
|------|-------|-------|---------------------|
| Dominant (60%) | `--bg` | `#fafafa` | Page background |
| Secondary (30%) | `--bg-surface` | `#ffffff` | Hero card background, week day cards, skeleton card |
| Accent (10%) | `--brand` | `#005696` | See reserved-for list below |
| Destructive | `--state-danger` | `#c62828` | Error card border, retry button background |

Accent (`--brand` / `#005696`) reserved for:
- Hero card left border accent (3px solid)
- "Find your store" primary CTA button background
- Week strip day name text color
- Store picker trigger in SharedNav
- Rarity badge "rare" tier background uses `--rarity-rare` (`#1565c0`), not brand accent

Additional semantic colors used (existing tokens, no new colors):
- `--rarity-ultra-rare` (`#7b1fa2`) -- ultra-rare badge background
- `--rarity-uncommon` (`#2e7d32`) -- uncommon badge background
- `--state-danger-bg` (`#fce4ec`) -- error card background
- `--text-muted` (`#666`) -- description text, store name
- `--text-subtle` (`#999`) -- freshness timestamp, secondary meta

## Component Inventory

Components used in this phase, referencing existing CSS classes.

### Hero Card (returning user)
- **Class chain:** `.card.card--hero.today-card`
- **Layout:** Flexbox row (`display: flex; gap: var(--space-4)`)
- **Left column:** 120px cone art container (`.today-flavor-cone.cone-lg`)
- **Right column:** Flavor name (`.today-flavor-name`), description (`.today-flavor-desc`), rarity badge (`.rarity-badge`), action CTAs via `actionCTAsHTML()`
- **Meta footer:** Store name + freshness timestamp (`.today-meta`), separated by `border-top: 1px solid var(--border-light)`
- **States:** confirmed (solid brand border), predicted (dashed border), no-data (gray border, "No data yet" text)

### Action CTAs (inside hero card)
- **Generator:** `actionCTAsHTML()` from `planner-ui.js`
- **Actions:** Directions, Set Alert, Subscribe (calendar .ics)
- **Link class:** `.cta-link` (delegated telemetry binding)
- **Layout:** Inline, wrapping, spaced by existing CTA row styling

### Loading Skeleton
- **Class:** `.today-card-skeleton` on a `.card` container
- **Contents:** 4 `.skeleton-line` divs (short, long, medium, short) with pulse animation
- **Purpose:** Occupies exact hero card dimensions to prevent CLS (HOME-03)
- **Enhancement:** Add a 120px x 140px gray rectangle on the left to match cone art area, preventing layout shift when cone loads

### Empty State (first visit)
- **Container:** `#empty-state` section
- **Content:** One sentence prompt + "Find your store" button + nearby store chips (if geo available)
- **Removed elements:** "New here?" kicker, "Custard Calendar tracks..." subtitle paragraph, 3-step guide, "View the map" secondary button, coverage disclaimer
- **New layout:** Centered single line of text + single primary button, with optional nearby store chips below

### Week-Ahead Section
- **Element:** `<details id="week-section">` (native HTML collapse, HOME-02)
- **Summary:** "Week Ahead" heading
- **Contents:** Horizontal scroll strip (`.week-strip`) of `.week-day-card` items (140px wide each)
- **Default state:** Collapsed (no `open` attribute)

### CTA Line (below fold)
- **Content:** Single text line "Get daily alerts" with inline link to `updates.html`
- **Style:** Body text (`--text-base`), link in brand color, no card wrapper
- **Replaces:** Full `.updates-cta-card` card with heading + paragraph + button

### Error State
- **Class:** `.card.card--danger.error-card`
- **Content:** Error message + "Try again" retry button (`.btn-primary.btn--danger.btn--sm`)

## Layout Contract

### Above the Fold at 375px

The viewport budget at 375px width, assuming standard mobile chrome (URL bar + status bar = ~130px visible), leaves approximately 530px of visible content height.

| Element | Height Budget | Notes |
|---------|--------------|-------|
| SharedNav bar | ~56px | Existing, not modified |
| Hero card | ~320px | 120px cone height + text + CTAs + meta + padding |
| Breathing room | ~154px | Ensures card bottom is comfortably visible |

The hero card MUST be the only content element visible without scrolling. No header (`<h1>Custard Forecast</h1>`), no subtitle, no decorative elements above it.

### Section Order (full page, top to bottom)

1. SharedNav (store indicator + nav links)
2. Hero card OR loading skeleton OR empty state OR error state (mutually exclusive)
3. Week-ahead `<details>` (collapsed by default)
4. CTA line ("Get daily flavor alerts")
5. Footer disclaimer

### Removed Sections

| Section | Current Element | Reason (from CONTEXT.md) |
|---------|----------------|--------------------------|
| Page header | `<h1>Custard Forecast</h1>` + `.header-subtitle` | "The card IS the forecast" -- strip it |
| Signals section | `#signals-section` | Serves power users who don't exist yet |
| Multi-store glance | `#multi-store-section` | Belongs on Compare page |
| Updates CTA card | `#updates-cta` `.updates-cta-card` | Replaced by single CTA text line |
| 3-step guide | `.first-visit-guide` | Replaced by minimal prompt |
| Coverage disclaimer | `.hero-coverage` | Removed from empty state |

## Interaction Contract

### State Transitions

```
[Page Load]
    |
    +-- localStorage has 'custard-primary'?
    |     YES --> Show skeleton (#today-loading)
    |             Fetch store manifest + flavor data
    |             On success --> Fade in hero card (#today-section)
    |             On error --> Show error state (#error-state)
    |     NO  --> Show empty state (#empty-state)
    |             Geo available? --> Show nearby store chips
    |             User taps "Find your store" --> Open SharedNav store picker
    |
[Store Selected (sharednav:storechange)]
    |
    +-- Hide empty state
    +-- Show skeleton
    +-- Fetch flavor data
    +-- On success --> Render hero card + show week-ahead + show CTA line
    +-- On error --> Show error state
```

### Transitions and Animation

| Trigger | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Skeleton to hero card | Opacity fade from 0 to 1 | 200ms | ease-out |
| First store selection (empty to hero) | Empty state fades out, skeleton appears, then hero fades in | 200ms each | ease-out |
| Error retry tap | Error state hidden, skeleton shown, re-fetch | immediate | -- |
| Week-ahead expand | Native `<details>` expand | browser default | -- |

No custom scroll animations. No parallax. No decorative transitions.

### CLS Prevention (HOME-03)

The skeleton MUST match the hero card dimensions exactly:
- Same padding: `1.25rem var(--space-5)` (matching `.today-card`)
- Same border-radius: `12px`
- Same border-left: `3px solid var(--border)` (gray during loading, brand color after load)
- Include a 120px-wide gray block on the left to reserve cone art space
- Skeleton lines on the right reserve text content space

This ensures zero layout shift when real content replaces the skeleton.

## Spacing Scale

Declared values from existing `style.css` custom properties (rem-based, converted to px at 16px base). Phase 34 uses only tokens from the standard set (multiples of 4 within 4, 8, 16, 24, 32, 48, 64):

| Token | Value | CSS Variable | Usage |
|-------|-------|-------------|-------|
| xs | 4px | `--space-1` (0.25rem) | Icon gaps, inline padding |
| sm | 8px | `--space-2` (0.5rem) | Compact element spacing, button padding, share button gaps |
| lg | 16px | `--space-4` (1rem) | Default card padding, section spacing |
| xl | 24px | `--space-5` (1.5rem) | Hero card padding |
| 2xl | 32px | `--space-6` (2rem) | Header vertical padding |

Note: The codebase defines `--space-3` (12px / 0.75rem) as a pre-existing token. Phase 34 does not introduce or require this value. All Phase 34 components use `--space-2` (8px) or `--space-4` (16px) where intermediate spacing is needed.

Exceptions: OG card SVG/PNG layout uses pixel values directly (80px left margin, 60px top padding, 40px inter-element gap) because satori renders from absolute pixel dimensions, not CSS custom properties.

### OG Card Palette (Worker)

| Role | Value | Usage |
|------|-------|-------|
| Background gradient start | `#1a1a2e` | Top of card (existing pattern) |
| Background gradient end | `#16213e` | Bottom of card (existing pattern) |
| Accent bar | Dynamic per flavor via `BASE_COLORS[profile.base]`, fallback `#005696` | 8px bar at top of card |
| Primary text | `#ffffff` | Headline, archetype name |
| Secondary text | `#9EC5E8` | Flavor name, fact text |
| Tertiary text | `#4a4a5a` | Footer URL, CTA text |
| Alternate gradient (trivia/rarity) | `#003A6B` to `#005696` | Flavor rarity cards use the diagonal brand gradient (matches existing trivia cards) |

**Quiz OG Card Decision (Claude's Discretion):** Use the dark-to-navy gradient (`#1a1a2e` to `#16213e`) as the base for quiz result cards. The accent bar color is dynamically set from the matched flavor's `BASE_COLORS` profile. This keeps all card types visually consistent rather than introducing quiz-mode-specific accent colors (which only exist on the frontend and would not carry meaning to social platform viewers who have never seen the site).

**Flavor Rarity OG Card Decision (Claude's Discretion):** Use the brand diagonal gradient (`#003A6B` to `#005696`) matching the existing trivia card pattern, since rarity cards are a "did you know?" style factoid, not a daily forecast.

## Interaction Contracts

### Skip-to-Result Mode (quiz.html?archetype=X&flavor=Y)

1. On page load, engine.js reads `archetype` and `flavor` URL params
2. If both params present and archetype ID matches a known archetype:
   - Quiz form section (`.quiz-panel`) is hidden
   - Quiz hero section (`.quiz-hero`) is hidden
   - Result section (`#quiz-result`) is shown immediately with the archetype/flavor data
   - Share button mounts with flavor-themed share text
   - "Take the quiz yourself" CTA button renders below the result card as `.btn-primary`; clicking it clears URL params and reloads quiz form
3. If archetype param does not match any known archetype:
   - Show empty state message in result section
   - "Take the quiz yourself" CTA button still renders

**Loading/Transition Decision (Claude's Discretion):** No special loading animation. The result renders synchronously from the archetype JSON data (already bundled in the page). Cone art loads asynchronously via `renderHeroCone()` as it does for normal quiz results. This avoids adding new animation code for a path that is inherently fast.

### Radar Flavor Share Icons

1. Each flavor entry in the Radar list gets a small share icon button
2. Icon: inline SVG share/arrow-up icon (12x12px), styled as `.btn--icon` variant at 28x28px touch target
3. Position: right-aligned within the flavor entry row, vertically centered
4. Tap triggers `navigator.share()` (mobile) or clipboard copy (desktop) with flavor-themed text
5. Clipboard confirmation: icon briefly changes to a checkmark for 2 seconds, then reverts

**Share Icon Design Decision (Claude's Discretion):** Use the iOS-style "square with arrow up" share icon at 12px, rendered as inline SVG. This is the most universally recognized share affordance on mobile (the primary target platform). The icon inherits `color: var(--text-muted)` at rest and transitions to `color: var(--brand)` on hover/active.

### Radar Flavor Highlight (arriving from shared link)

1. When `?flavor=X` param is present on index.html (via radar.html redirect)
2. Page scrolls to the matching flavor entry using `scrollIntoView({ behavior: 'smooth', block: 'center' })`
3. Matching entry receives a highlight ring: `box-shadow: 0 0 0 3px var(--brand)` with `border-radius: 8px`
4. Highlight fades after 3 seconds via CSS transition: `box-shadow` transitions from `0 0 0 3px var(--brand)` to `0 0 0 0 transparent` over 1 second

**Highlight Animation Decision (Claude's Discretion):** A simple expanding box-shadow pulse that fades out. No keyframe animation, no background color change. The box-shadow approach works on any element shape and does not alter layout. The 3-second visible duration gives the user time to orient, and the 1-second fade-out prevents an abrupt disappearance.

## Crawler Interception Contract

### User-Agent Detection

Social crawler regex: `/facebookexternalhit|Twitterbot|LinkedInBot|WhatsApp|Slackbot|Discordbot|TelegramBot/i`

### Quiz Shared URL: `quiz.html?archetype=X&flavor=Y`

When a social crawler requests this URL:
- Worker returns minimal HTML (not the full quiz page)
- `og:title`: `"{Archetype Name}: {Flavor Name} -- Custard Personality Engine"`
- `og:image`: `https://custard.chriskaschner.com/og/quiz/{archetype-id}/{flavor-slug}.png`
- `og:url`: The full shared URL
- `twitter:card`: `summary_large_image`
- `meta http-equiv="refresh"`: Redirects human browsers to the real page (safety net; crawlers ignore this)

### Flavor Shared URL: `index.html?flavor=X` (or `radar.html?flavor=X`)

When a social crawler requests this URL:
- Worker returns minimal HTML
- `og:title`: `"{Flavor Name} -- {Rarity Label}"`
- `og:image`: `https://custard.chriskaschner.com/og/flavor/{flavor-slug}.png`
- `og:url`: The full shared URL
- `twitter:card`: `summary_large_image`

### Non-Crawler Requests

Pass through to GitHub Pages as normal. No interception.

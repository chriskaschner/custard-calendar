**5. Add flavorToSlug() function** (after scoopColor, before drawConeIcon):
```javascript
// Convert flavor name to PNG filename slug, resolving aliases first
function flavorToSlug(name) {
  if (!name) return null;
  var key = name.toLowerCase().replace(/[\u00ae\u2122]/g, '').replace(/[\u2018\u2019]/g, "'").replace(/\s+/g, ' ').trim();
  var canonical = FLAVOR_ALIASES[key];
  var nameForSlug = canonical || key;
  var slug = nameForSlug.replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '');
  return slug || null;
}
```
This mirrors the cone-renderer.js heroConeSrc logic: normalize -> alias resolve -> kebab-case. The social-card.js version skipped aliases because the Worker already has canonical names, but the Scriptable widget receives raw flavor names from the API that may be variant forms.

**6. Add getConeImage() async function** (after flavorToSlug, before drawConeIcon):
```javascript
// Try loading L5 AI PNG from CDN; fall back to drawConeIcon on failure
async function getConeImage(flavorName, size) {
  var slug = flavorToSlug(flavorName);
  if (slug) {
    try {
      var url = CONE_PNG_BASE + "/" + slug + ".png";
      var img = await new Request(url).loadImage();
      return img;
    } catch (e) {
      // Network error or 404 -- fall through to drawConeIcon
    }
  }
  return drawConeIcon(flavorName, size);
}
```
Use Scriptable's `Request.loadImage()` which returns a Promise<Image> and throws on HTTP errors. Do NOT use `Image.fromURL()` -- that is not a real Scriptable API. The correct pattern is `new Request(url).loadImage()`.
